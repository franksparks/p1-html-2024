Fundamentos

1. Basic Git

Notas de la sesión

Git

  • Git es un sistema de control de versiones distribuido.
  • En un repositorio, podemos descargar el archivo ZIP que contiene la última versión publicada. No obstante, este no contiene los pasos intermedios.
  • Es ideal clonar el repositorio que sea de nuestro interés.
  • El remote es la fuente de la verdad.

Comandos útiles

  • git clone "URL-del-repositorio" - clona el repositorio
  • git init . - Inicializa git en el directorio actual.
  • git checkout -b "nombre-de-la-rama" - crea una nueva rama en git.
  • git status - muestra el estado actual del repositorio de trabajo.
  • git commit -m "mensaje" - confirma los cambios en staging.
  • git push - envía los cambios al repositorio remoto.
  • git log - muestra el historial de commits en el repositorio.

2. HTML and CSS I

Notas de la sesión

HTML

  • HTML (HyperText Markup Language) es un lenguaje de marcado estándar utilizado para crear y diseñar páginas web y aplicaciones web.
  • La sección HEAD contiene la metainformación.
  • La sección BODY contiene el cuerpo de nuestra web.
  • Usaremos las DevTools del navegador para analizar el DOM (Domain Object Model).
  • El Visual Studio Code es un browser!
  • Hay que estructurar el documento HTML correctamente y no caer en la div madness, usando el tag adecuado para cada sección.

CSS

  • CSS (Cascading Style Sheets) es un lenguaje utilizado para definir la presentación y el aspecto visual de los documentos HTML.
  • Se recomienda usar las unidades CSS em y rem, ya que son relativas.
  • Reglas CSS: Selectores que apuntan a elementos HTML, especificando propiedades a modificar y valores a aplicar. Quién, qué, cómo.
  • Podemos definir varias reglas por etiqueta HTML.
  • Podemos especificar varios tags por regla.
  • Las marcas CSS han de usarse como sal y pimienta.

3. HTML and CSS II

Notas de la sesión

Contenido incrustado

  • Podemos definir diferentes fuentes a una imagen para usarlas en base a una condición (como el tamaño de la pantalla).
  • Los iframes nos permiten mezclar dos webs en una, incrustando una ventana en nuestra web.
  • Las imágenes SVG son ideales para iconos, ya que no se les especifica tamaño sino coordenadas.

Listas

Etiquetas HTML de las listas:

  • ol - Lista ordenada
    •   <ol>
      <li>First</li>
      <li>Second</li>
      <li>Third</li>
      <li>Fourth</li>
  </ol>
    1. First
    2. Second
    3. Third
    4. Fourth
  • ul - Lista sin orden
    •   <ul>
      <li>First</li>
      <li>Second</li>
      <li>Third</li>
      <li>Fourth</li>
  </ul>
    • First
    • Second
    • Third
    • Fourth
  • li - Elemento de la lista

Tablas

Etiquetas HTML de las tablas:

  • table - Etiqueta principal de la tabla
  • thead - Header de la tabla
  • tbody - Cuerpo de la tabla
  • th - Header de columna de la tabla
  • tr - Fila de la tabla
  • td - Información de la celda 
    •     <table>
      <thead>
        <tr>
          <th>Day of the week</th>
          <th>Mode</th>
        </tr>
      </thead>
      <tbody>
        <tr>
          <td>Tuesday</td>
          <td>In person</td>
        </tr>
        <tr>
          <td>Thursday</td>
          <td>Online</td>
        </tr>
      </tbody>
    </table>
    Day of the week Mode
    Tuesday In person
    Thursday Online

Flexbox

  • Flexbox nos permite organizar los elementos en el eje horizontal y vertical.
  • Lo activamos mediante la propiedad CSS display:flex.
  • Aplicando flex-direction:column, resultará en justify-content afectará al eje vertical y align-items al eje horizontal.
  • Aplicando flex-direction:row, resultará en justify-content afectará al eje horizontal y align-items al eje vertical.
  • flex-end y flex-start :
  • stretch y center :
  • Podemos expandir los elementos para que utilicen el espacio disponible en proporción al peso que les configuremos mediante flex: "peso-del-elemento".

Posicionamiento

  • position: static - Valor por defecto, posición natural del elemento.
  • position: relative - Nos permiten mover elementos en relación a su posición estática.
  • position: absolute - El elemento se posiciona respecto al elemento precedesor "relative" más cercano.
  • position: fixed - El elemento se posiciona respecto al documento HTML.
  • position: sticky - Combinación entre static y fixed.

Atributos globales

  • id="id-del-elemento" - Identifica a un elemento.
  • class="clase-del-elemento" - Identifica a un grupo de elementos.
  • style="estilos-a-aplicar" - Añade estilos al elemento.

Selectores

  • #"id-del-elemento" - Referencia a un id de un elemento.
  • ."clase-del-elemento" - Referencia a una clase de un grupo de elementos.
  • "tag"."clase-del-elemento" - Referencia una etiqueta HTML que contenga además la clase especificada.
  • "tag"["atributo"] - Referencia una etiqueta HTML que contenga el atributo especificado.
  • "tag"["atributo"] - Referencia una etiqueta HTML que contenga el atributo especificado, con el valor indicado.
  • "tag uno" + "tag dos" - Referencia al elemento a justo después de b.
  • "tag uno" > "tag dos" - Referencia al elemento a con el elemento b como ancestro directo.

4. HTML and CSS III

Notas de la sesión

Pseudo clases y elementos

Hacen referencia a elementos y clases que vienen identificados directamente por el navegador.

  • ::before - Pseudo elemento que identifica el elemento vacío anterior al elemento seleccionado.
  • ::before - Pseudo elemento que identifica el elemento vacío posterior al elemento seleccionado.
  • :first-child - Pseudo clase que identifica el primer descendiente de su ancestro.
  • :nth-child(n) - Pseudo clase que identifica un/unos descendiente/s de su ancestro, en base a una formula dada.

Variables CSS

  • Permiten almacenar valores para reutilizarlos. Ideal para definir paletas de colores.

La cascada

Cómo se priorizan las reglas CSS:

  1. Importance (the !important keyword).
  2. Origin (user-agent, user, author) and layers (@layer, @import).
  3. Specificity.
  4. Order of appearance.

Especificidad

Es un vector de cuatro números que nos permite analizar el peso de las diferentes reglas CSS.

  • elemento - (0,0,0,1)
  • clase - (0,0,1,0)
  • id - (0,1,0,0)
  • inline style - (1,0,0,0) 
    • h1                   /* (0, 0, 0, 1) */
p em                 /* (0, 0, 0, 2) */
.grape               /* (0, 0, 1, 0) */
*.bright             /* (0, 0, 1, 0) */
p.bright e.dark      /* (0, 0, 2, 2) */
#id216               /* (0, 1, 0, 0) */
div#sidebar *[href]  /* (0, 1, 1, 1) */
nav#menu li#top      /* (0, 2, 0, 2) */

La palabra reservada !important se impone al resto y generaría su jerarquía propia.

DOM (Domain Object Model)

El DOM es la representación interna en memoria de la estructura HTMl de la página.

El navegador lleva a cabo 4 tareas principales con un documento HTML:

  • Recuperar su texto (Fetch).
  • Analizar el texto (Parse).
  • Mostrarlo por pantalla.
  • Escuchar eventos que puedan producirse.

Mediante la variable document JavaScript nos permite manipular el DOM de múltiples maneras.

Element queries

  • getElementsByTagName("tag") - plural - Devuelve los elementos con el nombre especificado.
  • getElementById("id") - singular - Devuelve el elemento con el id especificado.
  • getElementsByClassName("class") - plural - Devuelve los elementos con la clase especificada.
  • querySelector(sel) - Devuelve el primer elemento que cumpla el criterio especificado.
  • querySelectorAll(sel) - Devuelve todos los elementos que cumplan con el criterio especificado.

5. TypeScript Basics

Notas de la sesión

TypeScript es una capa por encima de JavaScript. Proporciona un sistema de tipos muy potente.

  • TypeScript aporta una comprobación estática del programa en tiempo de compilación. También muestra posibles maneras de completar campos, métodos o parámetros.
  • Los tipos en TypeScript son immutables.
  • A la hora de definir funciones, necesitamos indicar el tipo de las variables.
  • El tipo del resultado de una función suele estar bien inferido por TypeScript, pero podemos especificarlo.
  • Podemos añadir ? a un parámetro para indicar que tal vez no lo recibamos. También podemos establecer un valor por defecto.

Buenas prácticas

Siempre comprobaremos que si algo es NaN mediante Number.isNan("elemento-a-comprobar")

Conversión a boolean en TypeScript

  • if (num) devuelve true si el número no es cero.
  • if (str) devuelve true si el string no está vacío.
  • if (obj) devuelve true si el objeto no es null o undefined.

6. Objects

Notas de la sesión

Objetos

  • Representación de un concepto y sus propiedades
  • Con el operador . accederemos a sus propiedades.
  • Podemos mutar un objeto en cualquier momento, añadiendo, editando o eliminando sus propiedades.
  • La notación [ ] nos permite acceder a campos con nombres que contienen puntos o espacios.

Accediendo a objetos

  • Object.keys(obj) devuelve array de campos
  • Object.values(obj) devuelve array de valores
  • Object.entries(obj) devuelve array de pares "campo-valor"

Iterar en objetos

Para iterar por las propiedades de un objeto, necesitamos la palabra reservada in: 

for(let field in object){
    console.log(field)
    console.log(agent[field])
}

Comprobar si existe una propiedad en un objeto

if("name" in agent){
    console.log(agent[name])
}

Abreviar declaración de propiedades de objetos

Si declaramos una variable y más tarde queremos usarla como propiedad de un objeto, no necesitamos especificar su valor de nuevo. 

let x = 3
let y = 5
let coord={
    x, y
}

TypeScript Object Types

  • Podemos especificar el tipo de dato para cada propiedad, aunque TypeScript ya lo infiera por sí mismo.
  • Podemos definir nuestras estructuras de tipos para reaprovecharlas. 
    • type coord2d ={
    x: number,
    y: number
}
  • Con el operador ? marcaremos campos como opcionales.
  • Podemos usar el operador | para especificar varios tipos posibles, incluyendo undefined.
  • Interface: Identifica una serie de propiedades como un elemento, pero obteniéndolas por métodos como los getter.
  • Podemos marcar propiedades como readonly para evitar modificaciones, solo en interfaces. 
    • let person : {
    name: string,
    readonly dni: string
}

Arrays

  • Secuencia de elementos, cada uno con un índice único.
  • a[0] - elemento en posición 0 del array a.

Iterar en arrays

Para iterar por los elementos de un array, necesitamos la palabra reservada of: 

for(let elem of array){
    console.log(elem)
}

Algunos métodos de arrays:

  • push() - Añade elemento al final.
  • pop() - Elimina el último elemento.
  • shift() - Elimina un elemento del principio.
  • unshift() - Añade un elemento al principip.
  • Array.isArray() - Permite comprobar si un elemento es un array de manera fiable.

TypeScript Array Types

  • Arrays con un tamaño fijo.
  • let dni: readonly [number, string] = [47745843, "M"]

7. Advanced Functions

Notas de la sesión

Funciones de primera clase

  • Podemos tratar a las funciones como cualquier otro tipo de variable.
  • Las funciones no tienen nombre. Su nombre proviene de la variable que apunta a la función.
  • Declararemos las funciones como const

Métodos

  • Las funciones definidas con la palabra reservada function tienen una variable especial, this.
  • Si hay un objeto a la izquierda de una llamada, el objeto se convierte en el this..
  • El contexto lo da la llamada.
  • Podemos forzar el vínculo en otra función mediante bind(). Aunque no haya objeto a la izquierda, el objeto vendrá informado por el vínculo generado artificialmente.
  • groucho.sayHi.bind(Chicco)

Arrow functions

  • Notación corta para definir funciones. Normalmente tienen una sola instrucción.
  • const arrowSuma2 = (x,y) => (x+y);
  • Si necesitamos más de una línea, podemos usar llaves.
  • No usamos la palabra reservada function, por lo que las arrow functions no tienen this
  • No podemos usar las arrow functions como métodos.

Arrow functions methods

Reciben item, index y array como parámetros.

Algunos métodos de Array son de alto orden porque reciben funciones como parámetro.

Esencialmente, tienen un bucle dentro. Recorren el array transformando el contenido de sus índices o tratándolos de algún modo.

  • map() - Aplica una función a cada elemento del array.
  • forEach() - Igual que map(), pero no produce un nuevo array.
  • filter() - Devuelve los elementos que cumplen una condición.
  • find() - Devuelve el primer elemento que cumple una condición.
  • findIndex() - Devuelve el índice del primer elemento que cumple una condición.
  • sort() - Ordena los elementos del array.
  • reduce() - Ejecuta una función acumulador sobre los elementos del array (sumador, máximo, mínimo)
  • some() - Devuelve true si alguno de los elementos del array cumplen una condición.
  • every() -Devuelve true si todos de los elementos del array cumplen una condición.

Closures / Contexto léxico

  • Jerarquía de entornos basados en llaves.
  • Clausura - Una función puede sobrevivir al ámbito en el que se creó.
8. Classes

Notas de la sesión

Clase: Definición de un tipo de Objeto. Contiene campos y métodos.

Constructor: Inicializa una instancia de clase, se llama implícitamente.

  • La palabra reservada new reserva memoria para el nuevo objeto de la clase.
  • Al añadir export a la declaración de la clase, ésta será visible desde fuera del módulo donde ha sido declarada.

Encapsulación

Restringimos el acceso a parte del programa para discernir mejor el origen e los errores/imprevistos.

  • Versión TypeScript - Seteamos los parámetros de la clase como private con la palabra reservada private. Se comprueba el acceso en tiempo de compilación.
  • Versión JavaScript - Seteamos los parámetros de la clase como private con # (también a las invocaciones del campo). Se comprueba el acceso en tiempo de ejecuión.

Al configurar un campo como privado, solo se puede acceder desde los métodos de la clase, lo que acota los posibles errores a esos métodos.

En JavaScript, podemos implementar getters y setters modernos, que limitan el acceso directo sin tener que cambiar los accesos en el resto del código.

  • console.log(person.fullName) - getter
  • person.fullName = "John Doe" - setter

Herencia

Relación entre clases con propiedades y métodos en común. La subclase tiene acceso a los recursos de la superclase.

class Dog extends Animal{}

La primera llamada de una clase derivada debe ser a super().

Las subclases pueden reescribir los métodos que heredan.

Interfaces

Se utiliza la palabra reservada implements.

Comunica partes del código. La parte del código que implementa la interfaz con la parte que la usa.

Una clase puede implementar varias interfaces a la vez.

instanceof

Devuelve un booleano en función de si una variable es de una clase concreta, más allá de los tipos base. También tiene en cuenta las superclases.

Excepciones

Dada una función que detecta un fallo, se recorre la pila de funciones buscando la función que esté mejor capacitada para lidar con el fallo.

throw new Error("----")

El código a proteger se envuelve en un try-catch. Se intenta ejecutar el código incluido en el try y si hay un error, se captura en el catch.

Podemos discriminar entre tipos de errores y devolver diferentes objetos error en base a éstos.
